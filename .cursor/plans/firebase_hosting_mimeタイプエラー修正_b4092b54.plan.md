# Firebase Hosting MIMEタイプエラー修正

## 問題の原因

`firebase.json`の`rewrites`設定で`"source": "**"`がすべてのパスにマッチするため、`/wasm/controller.js`へのリクエストも`/index.html`にリライトされています。その結果、JavaScriptファイルではなくHTMLが返され、MIMEタイプが`text/html`になってしまい、ブラウザがモジュールとして読み込めません。

## 解決策

`firebase.json`の`rewrites`設定を修正して、静的ファイル（`.js`、`.wasm`、`.css`、`.png`など）がリライトされないようにします。Firebase Hostingでは、実際のファイルが存在する場合は自動的にそのファイルが提供されますが、存在しない場合でも静的ファイルの拡張子を持つパスはリライトから除外する必要があります。

## 実装内容

### 1. `firebase.json`の修正

[backcast/firebase.json](backcast/firebase.json)の`rewrites`設定を修正します：

- 現在の設定：すべてのパス（`**`）を`/index.html`にリライト
- 修正後：静的ファイルの拡張子を持つパスを除外し、それ以外のパスのみを`/index.html`にリライト

具体的には、`rewrites`の`source`パターンを`**/*.{js,wasm,css,png,jpg,jpeg,gif,svg,ico,woff,woff2,ttf,eot,json}`以外のパスにマッチするように変更します。ただし、Firebase Hostingの`rewrites`では否定パターンが直接サポートされていないため、以下のアプローチを取ります：

1. **静的ファイルの拡張子を明示的に除外する方法**：`rewrites`の`source`を修正して、静的ファイルの拡張子を持つパスにマッチしないようにする
2. **または、`/wasm/**`パスを明示的に除外する方法**：`rewrites`の前に、`/wasm/**`パスに対する設定を追加する

最も確実な方法は、`rewrites`の`source`パターンを修正して、静的ファイルの拡張子を持つパスを除外することです。Firebase Hostingでは、実際のファイルが存在する場合は自動的にそのファイルが提供されるため、`rewrites`はファイルが存在しない場合のみ適用されます。しかし、`/wasm/controller.js`が実際には存在しない場合（動的に生成されるか、別の場所にある場合）、リライトを避けるために、`/wasm/**`パスを明示的に除外する必要があります。

## 修正内容

`firebase.json`の`rewrites`セクションを以下のように修正します：

```json
"rewrites": [
  {
    "source": "!/wasm/**",
    "destination": "/index.html"
  },
  {
    "source": "**",
    "destination": "/index.html"
  }
]
```

または、より一般的なアプローチとして、静的ファイルの拡張子を持つパスを除外します：

```json
"rewrites": [
  {
    "source": "**/*.{js,wasm,css,png,jpg,jpeg,gif,svg,ico,woff,woff2,ttf,eot,json}",
    "destination": "/index.html"
  },
  {
    "source": "**",
    "destination": "/index.html"
  }
]
```

ただし、Firebase Hostingの`rewrites`では否定パターン（`!`）がサポートされていないため、実際には以下のアプローチを取ります：**推奨アプローチ**：`rewrites`の順序を利用して、まず`/wasm/**`パスを除外し、その後でその他のパスをリライトします。しかし、Firebase Hostingでは否定パターンがサポートされていないため、代わりに以下のように設定します：

```json
"rewrites": [
  {
    "source": "**",
    "destination": "/index.html",
    "regex": "^(?!.*\\.(js|wasm|css|png|jpg|jpeg|gif|svg|ico|woff|woff2|ttf|eot|json)$).*$"
  }
]
```

しかし、Firebase Hostingの`rewrites`では`regex`フィールドがサポートされていないため、実際には以下のアプローチを取ります：**実際の解決策**：Firebase Hostingでは、実際のファイルが存在する場合は自動的にそのファイルが提供されます。問題は、`/wasm/controller.js`が存在しない場合に`index.html`にリライトされてしまうことです。最も簡単な解決策は、`rewrites`の`source`パターンを修正して、`/wasm/**`パスを明示的に除外することです。しかし、Firebase Hostingでは否定パターンがサポートされていないため、代わりに以下のように設定します：

1. `rewrites`の前に、`/wasm/**`パスに対する設定を追加する（ただし、これはリライトを除外する方法ではない）
2. または、`headers`セクションで`/wasm/**`パスに対する設定を追加し、リライトを回避する

実際には、Firebase Hostingの動作を理解する必要があります：
